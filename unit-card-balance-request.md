# 機体カード性能設計の相談用テキスト

以下は、別AIにそのまま渡して相談するための依頼文です。

---

## 依頼内容

ブラウザで遊ぶ小規模なSRPGを作っています。  
ガンダム風の「機体カード・武器カード・キャラカード・戦艦カード」を集めて部隊を強くしていくゲームにしたいです。

現在はプロトタイプで、編成、戦闘、地形、射程、弾数、戦艦補給、複数武器による連続攻撃などの土台があります。  
次に、機体カードの性能を「仮数値」ではなく、ゲーム内で役割が分かれるように整理したいです。

機体データのバランス案を作ってください。

## 目指したい方向

- 原作再現100%より、SRPGとして役割が分かれることを優先したい。
- 量産機にも使い道があり、エース機は強いがコストが重い形にしたい。
- 連邦とジオンで特色を出したい。
- 後からカードを増やしても、同じ基準で性能を決められるようにしたい。
- 機体、武器、キャラ、戦艦をカードとして集めるゲームにしたい。
- 画像やアイコンは最後に付ける予定なので、今は数値と役割を優先したい。

## 現在の主な戦闘ルール

- マップはマス目式。
- 地上マップと宇宙マップがある。
- 地形には水中、森林、砂漠、デブリ帯、障害物がある。
- 地形適性がない機体は移動力と回避が下がる。
- 飛行ユニットは地上の地形適性を無視できる予定。
- 障害物は移動不可で、通常射撃の射線を遮る。
- 一部の砲撃・投擲武器は障害物を無視できる。
- 武器ごとに射程、最低射程、EN消費、弾数がある。
- 1ターン中に、武器の数だけ別々に攻撃できる。
- そのため、命中率は高すぎず低すぎずにしたい。
- 戦艦は1部隊に1隻。撃沈されると即敗北。
- 戦艦は高耐久、低命中、低回避。隣接ユニットに補給・修理ができる。
- 編成は最終的に「機体・装備A・装備B・オプション・パイロット」のようなフルセットにしたい。
- ただし機体によって、手持ち武器を2つ持てる、1つだけ持てる、構造上まったく持てない、という差を作りたい。
- 手持ち武器が少ない機体は、その代わりに固定武装、特殊能力、地形適性、コストの安さなどで価値を出したい。
- 一部の特殊能力は、後付けできるオプションパーツカードとしても作りたい。

## 機体データの項目

機体カードは、現在このような項目を持っています。

```js
{
  id: "gm",
  name: "ジム",
  faction: "federation",
  cost: 120,
  armor: 250,
  energy: 70,
  agility: 16,
  mobility: 4,
  fixedWeaponIds: ["headVulcan", "beamSaber"],
  forbiddenWeaponKinds: ["beam"],
  weaponSlots: 2,
  optionSlots: 1,
  mapTypes: ["ground", "space"],
  movementType: "normal",
  terrainSuitability: { water: false, forest: true, desert: true, debris: true },
  specials: [],
  imagePath: ""
}
```

### 項目の意味

- `cost`: 編成コスト
- `armor`: HPのような耐久力
- `energy`: ビーム武器や特殊武器に使うEN
- `agility`: 回避に関わる機体性能
- `mobility`: 移動力
- `fixedWeaponIds`: 機体固定武装
- `forbiddenWeaponKinds`: 原作設定上必要な場合だけ禁止する携行武器種別。通常は省略
- `weaponSlots`: 手持ち武器を装備できる数。通常は2、特殊な機体は1または0
- `optionSlots`: オプションパーツを装備できる数。まずは1を基本にしたい
- `mapTypes`: 出撃できるマップ。`ground`, `space`
- `movementType`: `normal` または `flying`
- `terrainSuitability`: 特殊地形への適性
- `specials`: 特殊能力。例: `magnetCoating`, `iField` など

## 装備スロットの考え方

最終的な1ユニットの構成は、以下のように考えています。

```text
機体カード + 装備A + 装備B + オプションパーツ + パイロット
```

ただし、すべての機体が装備A・装備Bを両方使えるわけではありません。

- 標準的なMS: 手持ち武器2枠
- 簡易量産機、支援機、一部の特殊機体: 手持ち武器1枠
- ボール、ガンタンクなど構造上手持ち武器を持てない機体: 手持ち武器0枠
- 手持ち武器0枠または1枠の機体は、固定武装や安いコストで価値を出したい
- 固定武装が多い機体は、手持ち武器が少なくても1ターン中の攻撃回数が多くなる可能性がある

## オプションパーツカードの考え方

将来的に、機体へ後付けできるオプションパーツカードも作りたいです。

例:

- マグネット・コーティング: 運動性や回避を少し上げる
- 増加装甲: armorを上げるがagilityやmobilityを少し下げる
- 推進剤タンク: mobilityや宇宙適性を補助する
- 水中戦装備: 水中適性を補う
- 追加ジェネレーター: energyを上げる
- 予備弾倉: 実弾武器の弾数を増やす
- 簡易Iフィールド: ビームダメージ軽減などの特殊能力を弱めに付与する

オプションパーツは、強力すぎると機体個性を消してしまうので、基本的には「弱点補助」か「方向性の強化」にしたいです。

## 出してほしいもの

以下の形式で提案してください。

1. まず、全体のバランス方針を短く説明してください。
2. 機体ごとの役割を説明してください。
3. 既存機体の再調整案を表で出してください。
4. 追加候補機体の性能案も表で出してください。
5. 必要なら、固定武装や原作設定上必須となる携行武器制限の見直し案も出してください。
6. 機体ごとの手持ち武器スロット数も提案してください。
7. オプションパーツカードの候補と効果案も提案してください。
8. 強すぎる、弱すぎる、役割が被るなどの注意点も書いてください。

## 表の形式

以下の列でお願いします。

| id | name | faction | cost | armor | energy | agility | mobility | weaponSlots | optionSlots | mapTypes | movementType | terrainSuitability | fixedWeaponIds | forbiddenWeaponKinds | specials | 役割メモ |
|---|---|---|---:|---:|---:|---:|---:|---:|---:|---|---|---|---|---|---|---|

`terrainSuitability` は以下のように書いてください。

```js
{ water: false, forest: true, desert: true, debris: true }
```

## 注意点

- キャラ補正もあるので、機体だけで命中・回避が完結しすぎないようにしたいです。
- 連続攻撃が可能なので、武器を多く持てる機体はそれだけで強くなります。
- 固定武装が多い機体は、手持ち武器なしでも行動回数が増えます。
- 手持ち武器スロットが2つある機体は、1ターン中の選択肢と攻撃回数が増えるので、コストや本体性能で調整したいです。
- 手持ち武器スロットが0または1の機体は、固定武装、特殊能力、地形適性、低コストなどで補いたいです。
- オプションパーツは機体の弱点を完全に消しすぎないようにしたいです。
- 低コスト機は弱くても、複数出せる・武器を選べる・地形で活躍するなどの価値が欲しいです。
- 高コスト機は強くてよいですが、撃墜された時の損失が大きくなるようにしたいです。
- 水中専用、地上専用、宇宙専用などの機体も後で作りたいです。
- 公式の台詞や文章を引用する必要はありません。

## 最終的に欲しい答え

「このゲームの機体カード性能の基準表」と「既存・追加候補機体の具体的な数値案」が欲しいです。  
実装しやすいように、できるだけ表形式で整理してください。
