# Gundam-style SRPG Prototype

仕様書をもとにした、学習用の小さなブラウザゲームです。

## 遊び方

1. 勢力を選びます。
2. 戦艦、艦長、副長、MS、キャラクター、手持ち武装を選んで部隊に追加します。
3. コスト上限内で出撃します。
4. 自軍ユニットを選び、緑のマスへ移動します。
5. 敵ユニットをクリックして攻撃対象にし、武装ボタンで攻撃します。
6. 武器ごとに攻撃できます。未使用の武器が残っていれば、同じ敵にも別の敵にも続けて攻撃できます。MSは3回目以降の攻撃で命中率が段階的に低下します。
7. 行動が終わったらターン終了を押します。
8. 戦艦に隣接してターンを終えると、EN・実弾・装甲・盾が少し回復します。
9. 戦艦は移動・攻撃できますが、撃沈されると即敗北です。
10. 戦艦には艦長・副長の2人を乗せられます。
11. キャラクターはMSにも戦艦にも配置できますが、同じ人物は1人だけです。
12. 「詳細」を開くと、機体・戦艦・武装・キャラクターの細かいステータスを確認できます。
13. マップには地上・宇宙があり、水中・森林・砂漠・デブリ帯では適性がない機体の移動力と回避率が下がります。
14. 障害物マスは移動できず、通常武器の射線を遮ります。クラッカーや一部砲撃武器は障害物を越えて攻撃できます。

## キャラクターの配置と能力

キャラクターはMSにも戦艦にも配置できますが、配置先によって使われるステータスやスキルが異なります。

- MS搭乗時は、主に射撃・格闘・反応・覚醒と、MS向けスキルが機能します。
- 戦艦の艦長時は主に指揮、副長時は主に支援・整備が機能します。
- スキルには「MS搭乗時のみ」「戦艦乗員時のみ」「どちらでも有効」のものがあります。カード詳細の効果説明を確認してください。
- 得意役割と異なる場所にも配置できますが、その場合は一部の能力やスキルが働かないことがあります。

## ファイルの役割

- `index.html`: 画面の土台です。
- `styles.css`: 見た目を決めています。
- `main.js`: ゲームのルールと画面更新を書いています。
- `data/game-data.js`: ゲームが実際に読むMS、戦艦、武装、キャラクターのデータです。
- `data/game-data.json`: 同じ内容をJSON形式で置いた参考データです。
- `CARD_DATA_GUIDE.md`: カードやステージを追加する時の手引きです。
- `assets/ms-token.svg`: マップ上の駒に使う仮シルエットです。

## 最初に学ぶとよい順番

1. `data/game-data.js` を見て、MSや武装の数値を少し変えてみる。
2. `main.js` の `hitRate` を見て、命中率がどう計算されるか読む。
3. `main.js` の `attack` を見て、攻撃の流れを読む。
4. `main.js` の `applyBattleshipSupport` を見て、戦艦補給の流れを読む。
5. `main.js` の `renderSetup` と `renderBattle` を見て、データが画面になる流れを読む。

この順番なら「データ」「計算」「画面」の3つを分けて理解しやすいです。

## 調整しやすい数字

- `HIT_RATE_BONUS`: 全体の命中率を底上げします。
- `MIN_HIT_RATE`: 最低命中率です。
- `data/game-data.js` の `battleships`: 戦艦の性能、コスト、補給量です。
- `data/game-data.js` の `characters`: MSにも戦艦にも配置できるキャラクターです。
- `data/game-data.js` の `weapons`: 武器の威力、命中、最小射程、最大射程、使用可能勢力、障害物越し攻撃可否です。
- `data/game-data.js` の `maps`: 地上・宇宙マップと特殊マスの配置です。
- `data/game-data.js` の `mobileSuits`: `mapTypes` と `terrainSuitability` で出撃可否や地形適性を調整できます。
- `main.js` の `hitRate` / `evasion`: MSと戦艦でキャラクター能力の反映方法を変えています。

## キャラ重複ルール

キャラクターには `characterKey` があります。
同じ `characterKey` のキャラは、別性能データとして追加しても同時に編成できません。

## データを増やした後の確認

カードやステージを追加したら、次の順で確認します。

1. `node work/check-gundam-prototype.js`
2. `node work/sync-game-data-json.js`

1つ目で存在しないID、装備枠オーバー、地形データの長さなどを確認します。
2つ目で `data/game-data.json` を `data/game-data.js` と同じ内容に同期します。
